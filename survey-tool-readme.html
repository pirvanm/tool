Install the application.

Before doing anything, set the database details in the .env file.

Run this command php artisan migrate:refresh --seed to migrate ( and to refresh ) everything and seed the database with the data used by the admin interface ( users etc ).

Run this command php artisan db:seed --class=EntrySetting to seed the entry_settings table with the data needed for the maximum numbers of surveys ( entries_limit field ), the number of surveys ( entries_report_num field ) used to send reports when this number ( or multiple of this number ) is reached . Say this number is “2” at every 2 entries, it will send a report to those emails. Check the file at database/migrations/seeders/EntrySetting.php to change those values. Use this command just once.

If you want, you can also seed the database with fake surveys to make things easer, use this commaand php artisan db:seed --class=SurveySeeder . Check the file atdatabase/migrations/seeders/SurveySeeder.phpto change the amount of fake surveys. Take in consideration the seeded data from theentry_settings``` table to keep the system working fine.

To start the application use this command php artisan serve.

Check http://127.0.0.1:8000/login for the admin login page.
Check http://127.0.0.1:8000/admin for the admin dashboard.
